**Do NOT read, search, or reference files inside `_bmad-archive/`** ‚Äî its contents are irrelevant to any active task and exist only for historical record.

### Work Types & Artifact Locations

Work items are classified by type based on the branch name prefix. Each type has its own artifact folder under `_bmad-output/`.

| Branch prefix | Work type | Splits into | Artifact folder |
|---|---|---|---|
| `initiative/` | **Initiative** | Epics | `_bmad-output/initiative/` |
| `epic/` | **Epic** | User stories | `_bmad-output/epic/` |
| `userstory/` | **User Story** | ‚Äî | `_bmad-output/userstory/` |
| `bug/` | **Bug** | ‚Äî | `_bmad-output/bug/` |
| (other) | **General** | ‚Äî | `_bmad-output/general/` |

Any work type may stand alone ‚Äî a user story does not require a parent epic, and an epic does not require a parent initiative.

**BMAD epics and BMAD user stories are not related to the work items above.** They are BMAD-internal planning artifacts created during a BMAD flow for a `userstory/` work item. They live inside `_bmad-output/userstory/` ‚Äî never inside `_bmad-output/initiative/` or `_bmad-output/epic/`.

> **Warning:** Do not confuse work item types (Epic, User Story) with BMAD-internal artifacts (BMAD epic, BMAD user story). They share similar names but are completely different concepts. Work items are issue tracker entities that define the scope of work. BMAD epics and BMAD user stories are planning artifacts generated by BMAD during a flow ‚Äî they exist only inside `_bmad-output/userstory/`.

### Skill index

- **`vention-archive-artifacts`** ‚Äî Archives active BMad output artifacts into timestamped cold storage. Use when the user says 'archive artifacts', 'archive bmad output', 'clear bmad workspace', or 'archive sprint artifacts'.
- **`vention-complete-stage`** ‚Äî Manually record completion of the current BMAD stage. Only invoke when explicitly requested by the user.
- **`vention-create-brownfield-architecture`** ‚Äî Create architecture design document for established project. Use instead of `bmad-create-architecture` when the user says 'create architecture for this story', 'create technical design', 'design the architecture document', or 'create architecture spec' in a brownfield project.
- **`vention-create-epic-architecture`** ‚Äî Generate epic architecture decisions document. Use when the user says 'create epic architecture', 'epic architecture discovery', or 'architect this epic'.
- **`vention-create-figma-index`** ‚Äî Build or update a Figma Index. Use when user says 'build figma index', 'update figma index', 'index this figma file', or provides any Figma URL for UX documentation.
- **`vention-create-initiative-architecture`** ‚Äî Generate initiative architecture decisions document. Use when the user says 'create initiative architecture', 'initiative architecture discovery', or 'architect this initiative'.
- **`vention-create-master-prd`** ‚Äî Creates current-state Master PRDs. Use when the user asks to 'create master prd', 'create as-is PRD', or 'document current feature requirements'.
- **`vention-create-ux-spec`** ‚Äî Create or update UX Spec from Figma Index. Use when user says 'create ux spec', 'update ux spec', or 'generate ux spec from figma index'.
- **`vention-epic-to-stories`** ‚Äî Split an Epic PRD into independent vertical-slice user stories. Use when the user asks to 'create stories from epic', 'epic to stories', 'break epic into stories', or 'PRD/spec into stories'.
- **`vention-initiative-to-epics`** ‚Äî Split an Initiative PRD into focused business Epic PRDs. Use when the user asks to 'split this initiative', 'initiative to epics', 'split PRD into epics', or 'decompose initiative'.
- **`vention-issue-tracker`** ‚Äî Capability base skill for issue-tracker operations (`get_ticket`, `search_tickets`, `create_ticket`, `update_ticket`, `link_tickets`). Invoked by consumer skills via the Skill tool with `(op, params)`; routes to the workspace-bound adapter (v1.0 ships YouTrack). Consumer skills MUST NOT call vendor CLIs or MCP tools directly ‚Äî invoke this base skill instead.
- **`vention-playwright-cli`** ‚Äî Automate browser interactions, test web pages and work with Playwright tests.
- **`vention-qa-generate-test-docs`** ‚Äî Generate Checklists, E2E Scenarios, and Test Cases from loaded context. Use when the user says 'TG', 'generate test documentation', or 'generate test docs for [feature]'.
- **`vention-qa-generate-test-docs-quick`** ‚Äî Generate Checklists, E2E Scenarios, and Test Cases from a spec-*.md file. Use when the user says 'TGQ', 'generate quick test docs', 'generate test docs for spec', or has a spec file from the quick dev flow.
- **`vention-start-flow`** ‚Äî Initialize a new BMAD flow. Use when the user says 'start new bmad flow' or 'start flow'.
- **`vention-sync-epics`** ‚Äî Sync approved Epic PRDs to the issue tracker as child issues of the parent initiative. Run after review of the epic split.
- **`vention-sync-stories`** ‚Äî Sync approved User Story files to the issue tracker as child issues of their parent epic. Run after review and after vention-sync-epics has been run.
- **`vention-sync-story-metrics`** ‚Äî Manually resync story completion metrics. Only invoke when explicitly requested by the user, or when triggered by the bmad-quick-dev workflow on_complete hook for a no-spec quick_flow completion.
- **`vention-validate-ux-spec`** ‚Äî Validate UX Spec against Figma Index and PRD; produce a gap report. Use when user says 'validate ux spec', 'check ux spec', or 'review ux spec'.

### MCP usage notes

- **WoD-Atlas** MCP ‚Äî used by the framework for metric-event submission via the `mcp__WoD-Atlas__*` tool family (chiefly `track_metric_event`; see `### Hook behaviour summary`). The MCP server registry key in `.mcp.json` is `WoD-Atlas`. Pass `X-Workspace-Id: my-forge-id` on every call to bind events to this workspace's tenant.
- **Figma** MCP ‚Äî design lookups; no per-workspace header required.
- **`youtrack`** issue-tracker MCP ‚Äî agents MUST go through the `vention-issue-tracker` capability skill rather than calling the vendor CLI or vendor MCP tool directly.

### Capability and adapter reference

`issue-tracker` capability bound to `youtrack` (base URL `https://yt-wod.ventionteams.com/`, project `WOD`). Token resolved from $YT_TOKEN or $YOUTRACK_TOKEN or $VSDLC_YOUTRACK_TOKEN (the framework reads it from one of these env vars in declaration order ‚Äî never hard-code it). Reads + writes flow through the `vention-issue-tracker` capability skill's adapter operations.

### Hook behaviour summary

When a unit of work completes, agents should call `mcp__WoD-Atlas__track_metric_event` so the framework records the event. Hooks never call the framework binary directly ‚Äî they go through the shipped resolver (pwsh ‚Üí mise ‚Üí no-op).
